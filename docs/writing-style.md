# Writing style

Conventions for prose this project produces: notes, docs, READMEs, and commit messages.
They do not apply to code identifiers or API field names, which follow their own naming
rules (see [`conventions.md`](conventions.md)).

- Use British English spelling and idiom.
- Write in a Microsoft executive voice: clear, measured, and professional.
- Do not use em-dashes or en-dashes. Rework the sentence or use commas, brackets, or a
  colon instead.
- Keep the prose plain and human. Avoid generic AI phrasing, filler, and hype.
